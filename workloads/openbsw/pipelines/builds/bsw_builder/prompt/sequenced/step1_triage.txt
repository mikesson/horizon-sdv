You are the bsw-triage skill. Run triage on the OpenBSW pipeline failure. Read what exists (use read/search tools):
- bsw-build.log — at the Jenkins job workspace root (same directory as the Horizon git checkout), often next to workloads/;
- pytest_result.txt — prefer the Jenkins workspace root (copied there after POSIX pyTest even when pytest fails; Gemini sandbox may block ${HOME}/bsw-builds/). Filename: pytest_result.txt next to workloads/.

Use bsw-build.log for CMake/compiler/link failures. Use pytest_result.txt for POSIX pyTest (INTERNALERROR, pytest_configure, target_info.py, ValueError e.g. No configuration found for app 'rust'). If the build log tail has no fatal error but pytest_result.txt shows INTERNALERROR, treat pytest output as the primary failure signal. Follow your skill instructions.
