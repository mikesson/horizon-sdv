# Horizon SDV Workspace Rules

Welcome! This workspace contains the Horizon SDV deployment repository. Any AI assistant or developer working in this codebase must adhere to the following rules:

## 1. Hermetic & Isolated Execution
*   **Docker Mandatory**: Always use the containerized deployer script `./tools/scripts/deployment/container-deploy.sh` instead of the native `deploy.sh` for Terraform operations. This ensures complete tool consistency (Terraform 1.14.2, gcloud CLI) and prevents host workstation configuration drift.
*   **Non-Sudo Docker**: Interact with the Docker daemon using the rootless/non-sudo `docker` command. Ensure the executing user is added to the `docker` group.

## 2. Security and Credentials
*   **Zero Secrets in VCS**: Never commit or write sensitive keys, personal access tokens (PATs), or passwords directly into files that are checked into git.
*   **Public Repository Default**: Keep `scm_auth_method = "none"` in `terraform.tfvars` when working with the public repository `https://github.com/GoogleCloudPlatform/horizon-sdv` to avoid credentials management overhead.

## 3. DNS and Networking State
*   **Static A Records**: When using static A records (`sdv_dns_use_static_a_records = true`), remember that DNS records must be manually mapped on the DNS host provider once the public external Load Balancer IP is generated by Terraform.

## 4. Platform Coding Guidelines
*   **Documentation Integrity**: Always preserve existing comments and docstrings in `.tf` and `.sh` files unless specifically requested to update them.
*   **Git Identity**: Always verify that the local Git configuration uses the approved Git identity (`mikesson`).
